# Руководство по разработке

## Начало работы

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Настройте переменные окружения в `src/environments/environment.ts`

3. Запустите dev сервер:
   ```bash
   npm start
   ```

## Структура кода

### Компоненты

Все компоненты используют standalone подход:
- Нет NgModules
- Импорты объявляются в декораторе `@Component`
- Каждый компонент самодостаточен

### Управление состоянием

Используется паттерн Store с Angular Signals:
- `signal()` для хранения состояния
- `computed()` для производных значений
- Реактивные обновления UI

### Работа с данными

Repository Pattern:
- Абстракция доступа к данным
- Преобразование между форматами (TypeScript ↔ Supabase)
- Обработка ошибок

## Стилизация

### CSS переменные

Все цвета и размеры определены в `src/styles.scss` через CSS переменные:
- Легко изменять тему
- Поддержка темной темы
- Адаптивные размеры

### Адаптивность

Mobile-first подход:
- Базовые стили для мобильных
- Медиа-запросы для больших экранов
- Breakpoint: 768px

## Добавление новых функций

### Новая страница

1. Создайте компонент в `src/app/pages/`
2. Добавьте маршрут в `src/app/app.routes.ts`
3. Создайте стили в `.scss` файле

### Новый Store

1. Создайте файл в `src/app/features/[feature]/[feature].store.ts`
2. Используйте `@Injectable({ providedIn: 'root' })`
3. Используйте signals для состояния

### Новый Repository

1. Создайте файл в `src/app/core/repository/`
2. Инжектируйте `SupabaseService`
3. Реализуйте методы `getAll()` и `save()`

## Лучшие практики

1. **Типизация**: Всегда используйте TypeScript типы
2. **Комментарии**: Документируйте публичные API
3. **Ошибки**: Обрабатывайте ошибки явно
4. **Реактивность**: Используйте signals для состояния
5. **Производительность**: Используйте `OnPush` стратегию где возможно
